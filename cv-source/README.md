# Beckwitt CV and Application Materials

This repository contains David Beckwitt's source CV, evidence-controlled resume and cover-letter formats, application examples, and AI-readable CV exports.

## Core rule

Use `AGENTS.md` as the operating guide. Every resume or cover-letter claim must be supported by `data/cv_master.json`, the source CV, the teaching philosophy materials, a user-provided update, or the job posting when describing employer needs.

Do not state the Ph.D. as completed unless David confirms it. Use: `Ph.D. candidate, Physics, University of Missouri, expected May 2026`.

## Prerequisites

- Python 3.10 or newer.
- Optional: a LaTeX distribution with `latexmk` for compiling the printable CV.
- Optional: `make`. Windows users can run every command directly with Python.

Preferred locked setup:

```bash
python -m pip install uv
uv sync --extra dev
```

Pinned pip fallback:

```bash
python -m pip install -r requirements-dev.txt -e .
```

`uv.lock` is the scanner-recognized dependency lockfile. Refresh it with `uv lock` after `pyproject.toml` dependency changes.

See `docs/index.md` for the repository map.
See `docs/setup-windows-latex.md` for MiKTeX, Perl, and strict LaTeX validation on Windows.
See `CONTRIBUTING.md` for branch, PR, evidence-control, privacy, and skipped-check reporting rules.
See `docs/reproducibility-governance-status.md` for the latest status of the LaTeX validation bug, strict-mode error, and governance features.

## Common workflows

| Task | Direct command | Make target |
| --- | --- | --- |
| Install dev tools | `python -m pip install -r requirements-dev.txt -e .` | n/a |
| Generate AI-readable exports | `python scripts/export_cv_ai.py` | `make ai-export` |
| Convert an application draft to DOCX | `python scripts/application_docx.py draft.md final.docx` | n/a |
| Run default repository checks | `python scripts/check_repo.py` | `make check` |
| Run strict LaTeX checks | `python scripts/check_repo.py --require-latex` | `make check-strict` |
| Build printable CV | `python scripts/build_cv.py` | `make cv` or `make build` |
| Run tests | `python -m pytest` | `make test` |
| Run tests with coverage | `python -m pytest --cov=scripts --cov-report=term-missing` | `make coverage` |
| Run lint | `python -m ruff check .` | `make lint` |
| Check formatting | `python -m ruff format --check .` | `make format-check` |
| Apply formatting | `python -m ruff format .` | `make format` |
| Run type checking | `python -m mypy .` | `make typecheck` |
| Clean local build files | `python scripts/clean_build.py` | `make clean` |

Default repository checks skip only missing or incomplete optional LaTeX tooling. Privacy, evidence-source, JSON, export freshness, `.docx` readability, placeholder, reference-exclusion, and forbidden binary-artifact failures still fail normally. Use strict LaTeX mode when a full TeX stack is expected; on Windows setups where MiKTeX provides `latexmk` but Perl is missing, strict mode is expected to fail until Perl or the LaTeX toolchain is fixed.

Normal CI runs the default Python validation path and does not require TeX, MiKTeX, or Perl. Strict LaTeX CI is available as a manual workflow in `.github/workflows/strict-latex.yml`. Secret scanning runs Gitleaks against the current checkout, not full repository history.

## Generated files

`exports/Beckwitt_CV_AI.md`, `exports/Beckwitt_CV_AI.json`, and `exports/Beckwitt_CV_AI.txt` are generated by `python scripts/export_cv_ai.py`. Refresh them whenever `data/cv_master.json` changes.

`docs/compiled/Beckwitt_CV_Compiled.pdf` is generated locally by `python scripts/build_cv.py` when LaTeX is available. Compiled PDFs are ignored because they can expose stale or private content if not regenerated.

`python scripts/check_repo.py` regenerates exports in a temporary directory and compares normalized content. It normalizes only export dates and newlines, so real content drift still fails.

## Repository structure

- `main.tex`, `settings.sty`, `sections/`, `bib/`, `images/` - source LaTeX academic CV.
- `data/cv_master.json` - structured evidence source for AI-readable CV exports and application drafting.
- `exports/` - generated Markdown, JSON, and plain-text CV exports. These are intentionally simple, table-free, and reference-free.
- `applications/job_families/` - reusable resume and cover-letter formats for metrology, data science, scientific sales, teaching, and neutron/X-ray scattering roles.
- `applications/common/` - intake, evidence-matrix, resume, and cover-letter templates used before drafting a job-specific package.
- `applications/examples/` - prior role-specific text examples for AI review. Original binary application files are local/private artifacts and are not tracked.
- `outputs/` - destination for finished job-specific `.docx` application packages. Final documents should be simple, one-column, and AI-legible.
- `docs/architecture.md` - source-of-truth, data-flow, privacy, evidence-control, and validation notes.
- `docs/index.md` - compact navigation map for maintainers and agents.
- `docs/setup-windows-latex.md` - Windows MiKTeX, Perl, and strict LaTeX validation notes.
- `docs/reproducibility-governance-status.md` - current status of reproducibility, LaTeX validation, type checking, and governance work.
- `docs/compiled/` - local ignored compiled PDFs. Regenerate when LaTeX source changes.
- `scripts/application_docx.py` - converts temporary Markdown/plain-text drafts into simple `.docx` files and extracts `.docx` text for validation.

## Application workflow

1. Save the job posting or URL.
2. Complete `applications/common/job_posting_intake.md`.
3. Build an evidence matrix from `applications/common/evidence_matrix_template.md`.
4. Pick the closest job-family resume and cover-letter format.
5. Draft the application in `outputs/<company>/<role>/` and save final documents as `.docx`.
6. Run `python scripts/check_repo.py` and the quality checklist in `AGENTS.md` before sending.

## Notes on examples

The example files in `applications/examples/` are archived writing samples. Treat them as examples of tone and organization, not as automatically approved evidence. If an example includes a metric, credential, instrument, or outcome that is not in `data/cv_master.json` or the source CV, verify it before reusing it.
