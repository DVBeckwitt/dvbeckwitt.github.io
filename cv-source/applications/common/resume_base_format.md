# Resume Base Format

Use plain ATS-friendly formatting. Avoid columns, icons, graphics, photos, skill bars, unusual fonts, and dense academic prose.

## Header

David Beckwitt
Email | LinkedIn | GitHub | optional location

Do not include private phone numbers or addresses unless David provides them for the specific application.

## Targeted summary

Use 2 to 3 lines. Name the role family, strongest technical evidence, and one differentiator. Do not use generic adjectives.

## Skills

Group skills by employer need. Put the most relevant group first.

## Experience

Put the most relevant role first. Translate research into employer outcomes, but keep each claim evidence-based.

Bullet formula:

Verb + method/tool + technical object + result or purpose.

## Projects

Use when software, modeling, data analysis, or instrumentation work is central to the role.

## Publications or presentations

Include for scientific, academic, scattering, metrology, and research-heavy roles. Omit or compress for general industry roles unless credibility matters.

## Education

State: Ph.D. candidate, Physics, University of Missouri, expected May 2026.

## Awards

Use selectively. Include teaching awards for teaching roles and research/scattering awards for scientific roles.
